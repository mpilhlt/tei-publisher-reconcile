# AGENTS.md / Claude.md

Project: a **TEI Publisher v10** extension implementing the **Reconciliation Service API**
(OpenRefine reconciliation, spec **0.2 and 1.0**). The **server** is a new **Jinks profile**; the
**client** lives in the existing **`annotate`** profile and is kept **decoupled** (configurable
endpoint URL, no hard dependency on the server profile). Conformance is gated by a **locally-run**
reconciliation testbench.

This file is orientation only — durable facts about *this* repo plus copy-paste commands. It does
**not** repeat the testing procedure; that lives in the skill below.

---

## Use the testing skill

For **any** task that authors/iterates a Jinks profile,
- edits a `*-api.tpl.json` spec or `*-api.xql` handler,
- deploys with jinks-cli,
- hits `api/*` routes,
- checks a manifest / reconcile / suggest / preview / extend response,
- debugs CORS,
- validates against JSON Schema (find schemas in `./reconc-specs/{0.2,1.0-draft}/schemas`),
- runs Cypress, or runs the reconciliation testbench

**Use the `teipublisher-reconciliation-testing`.**
Do this even for terse asks like "test my reconcile endpoint" or "regenerate the app".

Skill location: `./skills/teipublisher-reconciliation-testing` (install where your agent discovers
skills). Its scripts are referenced below.

Decided strategy: **schema validation in Cypress every iteration** (the project's native harness),
with the **local** reconciliation testbench (0.2 and 1.0) as the final gate.

## Architecture (v10 = Jinks profiles, not a monolith)

TEI Publisher v10 apps are **generated by Jinks** from composable **profiles**. There is no single
`custom-api.json`; each profile contributes its own OpenAPI spec + roaster (router) handler
module + handler namespace prefix, and roaster mounts them all.

- **Server profile** `profiles/reconcile/`
  - `config.json` — manifest. Declares the API and dependencies:
    `depends: ["base10", "registers"]`
    `api: [{ "prefix": "reconc", "id": "http://teipublisher.com/api/reconcile",
             "path": "reconcile-api.xql", "spec": "reconcile-api.json" }]`
  - `modules/reconcile-api.tpl.json` — OpenAPI 3.0.3 spec (Jinks **template** → `reconcile-api.json`).
    Routes carry `operationId: "reconc:<fn>"`.
  - `modules/reconcile-api.xql` — handlers `declare function reconc:<fn>($request as map(*))`.
  - `test/cypress/e2e/api/reconcile.cy.js` — API tests (schema-validated).
- **Client** in `profiles/annotate/resources/scripts/annotations/` wired into the entity-authority
  UI (`templates/pages/annotation/annotate-authorities.html`, the person/place/org/work editors).
  Decoupled: configurable endpoint URL; do **not** add `reconcile` to annotate's `depends`.

Live routes: `http://localhost:8080/exist/apps/<abbrev>/api/reconcile`  # <abbrev> being the name of the generated app including the reconciliation profile. For demo or testing use `tp-reconc`
(manifest = GET here, reconcile = POST here).

## Commands (canonical for this setup)

```bash
SKILL=skills/teipublisher-reconciliation-testing
APP=tp-reconc                        # generated-app abbreviation
PROFILE=./reconcile                  # local profile dir

# 1. Platform up (podman; v10 publishes 8080:8080; admin password empty)
$SKILL/scripts/up.sh                 # runs existdb/teipublisher:10.0.0 and waits for readiness

# 2a. First time only: create the app from your profile(s)
jinks create "$APP" -c reconcile/config.json -s http://localhost:8080/exist/apps/jinks -u tei -p simple

# 2b. Dev loop: live-sync profile edits, then regenerate the app
$SKILL/scripts/watch-profile.sh "$PROFILE"    # leave running in the background
$SKILL/scripts/regenerate.sh "$APP"           # jinks update; add --all to check every file

# 3. Inner test loop
$SKILL/scripts/ad-hoc-xquery.sh path/to/snippet.xq                                   # layer 1
( cd "$APP-checkout" && npx cypress run )                                            # Cypress API tests
$SKILL/scripts/cors-check.sh "http://localhost:8080/exist/apps/$APP/api/reconcile"   # CORS

# 4. Conformance gate — LOCAL testbench, both versions
# git clone https://github.com/reconciliation-api/testbench.git && (cd testbench && npm i && npm start)
cd reconc-testbench && npm i && npm start
# point it at http://localhost:8080/exist/apps/$APP/api/reconcile ; repeat for 0.2 and 1.0

# Reset to a clean platform
podman rm -f teipub        # with no named volume this fully resets; otherwise also: podman volume rm exist-data
```

If a change doesn't take effect: confirm `jinks watch` synced it, then that you re-ran
`jinks update`. **`.tpl.*` files only apply after regeneration** (templates must expand). Confirm
the running code is the new code before debugging logic.

## Hard rules

- **follow a red/green tdd approach**
- **Don't hand-edit generated app files as the source of truth.** Edit **profile source** and
  regenerate. Jinks preserves unrelated local edits and reports conflicts, but source-of-truth is
  the profile.
- **Roaster does not resolve `$ref`** — keep each OpenAPI spec self-contained (or bundle before use).
- **CORS is central, not per-handler.** The base controller echoes the request `Origin` only if it
  matches `config:origin-whitelist` (localhost/127.0.0.1 by default). Local testbench + Cypress
  (`cy.api` uses a localhost Origin) work unchanged. **Do not add per-route CORS code.** Widen the
  whitelist only if you deliberately switch to the hosted testbench (a security-relevant change).
- **Advertise both versions** in the manifest `versions` and make each version's responses conform
  (query shape differs; `type_strict` removed in 1.0).
- **Definition of done is not "curl works"** — see below.

## Credentials & secrets

- jinks-cli default app user is `tei` / `simple`; use `admin` / empty for DBA-level operations and
  `xst`. Pass via flags/env, **never hardcode secrets** in committed files:
  `JINKS_SERVER` (default `http://localhost:8080/exist/apps/jinks`), `JINKS_USER`, `JINKS_PASS`;
  `EXISTDB_SERVER`/`EXISTDB_USER`/`EXISTDB_PASS` for the REST/ad-hoc scripts.

## Don't commit

- The eXist data volume / any `exist-data` dir; `node_modules/`; Cypress `videos/`, `screenshots/`,
  `downloads/`; generated `.json` from `.tpl.json`; built `.xar` artifacts.

## Definition of done

- [ ] Ad-hoc / XQSuite logic checks pass.
- [ ] Cypress API tests green: manifest (GET, no params) + reconcile (POST) + any sub-services.
- [ ] Responses validate against the reconciliation JSON Schemas for **both** 0.2 and 1.0.
- [ ] `cors-check.sh` green (localhost Origin echoed; preflight ok) for every route the testbench calls.
- [ ] **Local** testbench **0.2** and **1.0** run clean against the generated app (artifact saved).
- [ ] Re-run once after `jinks update --all` (or a recreated container) to prove no reliance on
      leftover DB state.

## Host / environment notes

Host is Linux (WSL/Ubuntu or native Arch) using **podman**, not docker.

- `podman` is drop-in for `docker` it is preinstalled. Cancel development if it isn't.
- Rootless podman (Arch & Ubuntu/WSL) publishes 8080 without extra privileges.
- SELinux `:z`/`:Z` volume labels only where SELinux is enforcing (check `getenforce`).
- WSL2: run everything inside the distro; `localhost:8080` is reachable there and from Windows.

## Values to confirm (defaults chosen — correct any that are wrong)

- **Generated-app abbrev** (`<abbrev>`) — defaulted `tp-reconc`. The TEI Publisher convention is a
  `tp-` prefix. Reason: matches example apps (`tp-serafin`, `tp-workbench`).
- **Server profile name / dir** — defaulted `reconcile`. Reason: matches the API and is short.
- **Handler prefix / api id / route** — `reconc` / `http://teipublisher.com/api/reconcile` /
  `…/api/reconcile`. Reason: short, self-documenting, mirrors `dts`/`registers` conventions.
- **`depends`** — `["base10", "registers"]`. Reason: `base10` is required; `registers` holds the
  person/place/org/work authorities a reconciliation service matches against. Drop `registers` if
  you reconcile against something else.
- **Image pin** — `:10.0.0` over `:latest`. Reason: reproducibility.
